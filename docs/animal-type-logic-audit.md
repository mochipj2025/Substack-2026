# 動物タイプ判定ロジック監査メモ

## 現状

`diagnosis.js` では、当初すべての診断結果が `lion` に固定されていた。
そのため、公式の動物占い・個性心理學で確認した動物タイプと一致しなかった。

```js
animal: "lion"
```

## 2026-06-18 の修正 1

ライオン固定を外し、暫定的に「公式で確認した動物タイプ」を入力フォームで選べるようにした。

対象：

- `diagnosis.html`
- `diagnosis.js`

これにより、公式結果がライオン以外の場合でも、結果ページへ正しい動物タイプを渡せる。

## 2026-06-18 の修正 2

手入力式をやめ、生年月日から60分類番号を計算し、12動物タイプへ変換する自動判定に変更した。

実装：

- `calculateJulianDayNumber(year, month, day)`
- `calculateCharacterNumber(dateValue)`
- `calculateAnimalType(dateValue)`
- `characterNumberToAnimal`

現在の考え方：

- 生年月日からユリウス日番号を計算する
- 1921-01-01 を `1` とする標準的な60日周期の説明に合わせたオフセット `+49` で60分類番号へ変換する
- 1-60の分類番号を12動物へマッピングする

## 12動物へのマッピング

`diagnosis.js` の `characterNumberToAnimal` に保持している。

12動物だけでなく、将来60分類の詳細名を出す場合は、同じ番号を使って拡張できる。

## 追加検証が必要なもの

このロジックは、60分類番号を使う実装として導入した。
ただし、2026-06-19 の再監査で、公式サイト `https://www.noa-group.co.jp/kosei/result` の結果欄と一致しないサンプルが見つかった。

前回の「公式照合サンプル」はページ全体の動物名出現数を数えており、結果欄以外のナビ・説明・関連記事を拾っていた。
公式結果欄は `<p class="yourName">` と結果画像の `alt` / `src` を見る必要がある。

## 2026-06-19 の結論

現在の `calculateCharacterNumber()` は、標準的な「1921-01-01を1として60周期」説明とは整合している。
しかし、個性心理學研究所公式ページの現在の結果欄とは複数サンプルで一致しない。

そのため、現状のまま「公式の動物占いそのまま」として公開するのは危険。
公式一致を優先する場合は、公式と同じ計算規則を特定するか、誕生日別の早見表データを倉庫化して `calculateCharacterNumber()` を置き換える必要がある。

照合したい項目：

- 12動物タイプ
- 60分類番号
- 境界日
- うるう年
- 日本時間の日付扱い

## 注意

数秘術は生年月日からライフパスナンバーを計算している。
動物タイプは生年月日から60分類番号を計算している。
