# 占いロジック監査メモ

監査日: 2026-06-19

## 結論

重大なミス候補は動物占いの自動判定。
`diagnosis.js` の現在ロジックは、標準的な60日周期説明とは合うが、個性心理學研究所公式ページの結果欄とは複数サンプルで一致しない。

数秘術、星座、血液型、五行質問紙は現状の実装としては大きな破綻は見つからなかった。
ただし、星座の境界日と五行の同点処理は仕様として明記した方がよい。

## 動物占い

実装箇所:

- `diagnosis.js`
  - `calculateJulianDayNumber(year, month, day)`
  - `calculateCharacterNumber(dateValue)`
  - `calculateAnimalType(dateValue)`
  - `characterNumberToAnimal`

現在の式:

```js
return ((julianDayNumber + 49) % 60) + 1;
```

この `+49` は、1921-01-01 を `1` とする60日周期説明とは整合する。
しかし公式ページの結果欄とは一致しない。

公式照合で見つかった不一致:

| 生年月日 | 公式結果欄 | 現在ロジック | 判定 |
| --- | --- | --- | --- |
| 1990-04-12 | 粘り強いひつじ | 黒ひょう | 不一致 |
| 2000-01-01 | 束縛を嫌う黒ひょう | 虎 | 不一致 |
| 2026-06-18 | 感情豊かな黒ひょう | 虎 | 不一致 |
| 1975-01-01 | 人間味あふれるたぬき | 黒ひょう | 不一致 |
| 1988-08-08 | 人気者のゾウ | こじか | 不一致 |
| 2026-12-31 | コアラのなかのコアラ | コアラ | 一致 |

次の対応候補:

1. 公式一致を優先するなら、公式と同じ早見表データを倉庫化して `calculateCharacterNumber()` を置き換える。
2. 標準60日周期を採用するなら、画面に「公式ページとは異なる場合があります」と注記する。
3. 公開前は最低でも20件以上の生年月日で公式照合テストを残す。

## 数秘術

実装箇所:

- `diagnosis.js`
  - `calculateLifePath(dateValue)`

現在の仕様:

- 生年月日 `YYYY-MM-DD` の数字を全て合計する
- 11、22、33 はマスターナンバーとして残す
- それ以外は1桁になるまで足す

例:

| 生年月日 | 計算 | 結果 |
| --- | --- | ---: |
| 1990-04-12 | 1+9+9+0+0+4+1+2 = 26 -> 8 | 8 |
| 2000-01-01 | 2+0+0+0+0+1+0+1 = 4 | 4 |
| 2009-09-09 | 2+0+0+9+0+9+0+9 = 29 -> 11 | 11 |

大きなミスは見つからない。
ただし、流派によって33の扱いが違うことがあるので、サイト内では「マスターナンバー11/22/33採用」と決めておく。

## 星座

実装箇所:

- `diagnosis.js`
  - `calculateZodiacSign(dateValue)`
- `animals.js`
  - `calculateZodiacFromMonthDay(monthDayValue)`
  - `getResultZodiac()`

現在の境界:

- 牡羊座: 03-21 から 04-19
- 牡牛座: 04-20 から 05-20
- 双子座: 05-21 から 06-21
- 蟹座: 06-22 から 07-22
- 獅子座: 07-23 から 08-22
- 乙女座: 08-23 から 09-22
- 天秤座: 09-23 から 10-23
- 蠍座: 10-24 から 11-22
- 射手座: 11-23 から 12-21
- 山羊座: 12-22 から 01-19
- 水瓶座: 01-20 から 02-18
- 魚座: 02-19 から 03-20

大きなミスは見つからない。
ただし本格的な西洋占星術では年・時刻・出生地によって境界日が変わるため、このサイトでは「一般的な太陽星座の固定境界」として扱う。

## 五行質問紙

実装箇所:

- `diagnosis.js`
  - `fiveElementQuestions`
  - `calculateFiveElementResult(formData)`

現在の仕様:

- 10問
- 各回答が `wood` / `fire` / `earth` / `metal` / `water` のどれかに1点
- 最多点の属性を結果にする
- 同点時は `wood > fire > earth > metal > water` の優先順

大きなミスは見つからない。
ただし、同点時に木が強く出やすいので、同点なら「主属性 + サブ属性」を出す方式の方が本格的。
