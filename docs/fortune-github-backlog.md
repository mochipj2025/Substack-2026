# 占い本格化 GitHub整理メモ

作成日: 2026-06-23

このメモは、ケツ印どうぶつ診断を本格的な占いサイトに育てるための GitHub Issue 化リスト。
実装前に迷わないよう、Milestone、Label、Issue候補をここにまとめる。

## Milestone案

### v0.1 信頼できる診断ロジック

目的: 生年月日から出る結果がズレない状態にする。

- 動物占いの60分類ロジックを代表ケースで照合する
- 星座の境界日仕様を明文化する
- 数秘のマスターナンバー扱いを固定する
- 五行質問紙の点数と同点処理を仕様化する

### v0.2 結果データ倉庫

目的: JS内の文章を分離し、占い本文を増やしやすくする。

- 動物12タイプの基本読みをJSON化する
- 五行5タイプの読みをJSON化する
- 数秘 1-9 / 11 / 22 / 33 の読みをJSON化する
- 星座12タイプの読みをJSON化する
- 血液型 A / B / O / AB / 不明の読みをJSON化する
- 参照元メモとオリジナル文の境界を `docs/` に保存する

### v0.3 本格結果ページ

目的: 結果ページを「当たってそう」「読みたい」と感じる構成にする。

- 結果上部の称号と一言サマリーを磨く
- 5分類分析を縦並びで読みやすくする
- 長所、短所、総合運、開運ポイントを追加する
- 人間関係、活動・仕事運、疲れた時の戻り方を追加する
- 組み合わせ読み 動物×五行 / 動物×数秘 / 五行×星座 を増やす

### v0.4 共有・保存体験

目的: 家族メンバーに見せやすく、あとで保存しやすくする。

- 結果レポート専用ページを整える
- PDF保存のレイアウトを磨く
- Image 2.0 Promptを結果カード / アイコン / Pet差分に分ける
- Promptコピーの成功表示をわかりやすくする
- SNSやグループに貼りやすい短い結果文を用意する

### v0.5 公開前QA

目的: スマホで気持ちよく使えて、主要ケースで破綻しない状態にする。

- 代表テストケースを固定する
- スマホ表示をトップからPDFまで確認する
- GitHub Pagesでキャッシュ込みの表示確認をする
- 画像、PDF、Promptコピーの動作確認をする

## Label案

- `fortune-logic`: 占い計算、境界日、点数処理
- `reading-data`: 診断文、JSONデータ、文章倉庫
- `result-page`: 結果ページUI、読み順、表示改善
- `sharing`: PDF、Prompt、SNS共有
- `qa`: テストケース、スマホ確認、公開前確認
- `docs`: 仕様、参照元メモ、運用メモ

## Issue候補

### 1. 動物占い60分類ロジックの照合

Label: `fortune-logic`, `qa`

やること:

- `docs/animal-type-logic-audit.md` の不一致サンプルを確認する
- 代表20件以上の生年月日で、60分類番号と12動物を照合する
- 照合済みケースを `docs/fortune-logic-audit.md` に追記する
- 仕様が確定したら `diagnosis.js` の計算ロジックを固定する

完了条件:

- 代表ケースで動物タイプがズレない
- 公式情報をそのまま転載せず、ロジック確認用メモとして残す
